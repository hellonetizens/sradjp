    Ruby は MatzLisp といわれるぐらい Lisp 色が強いので、Ruby が好きな人は Emacs を好む傾向はあると思う。
    Python はブロックの終わりを判断する識別文字がないので、インデントを自動でつける Emacs や IDE 向きじゃないかも。

    C# のようにほぼ VS で開発するのを前提にして作られている言語だと IDE を使うしかない。
