Twitterの「おすすめユーザー」に近隣ユーザーが表示され騒動に 42

ストーリー by hylom 2011年05月20日 18時10分
すでに周囲にばれている私に死角はない 部門より

あるAnonymous Coward 曰く、

Twitterの「おすすめユーザー」欄にて、IPアドレスを元に近隣のユーザーが表示されるのでは、という話が先日話題になっていた（Togetterまとめ）。

会社の同僚や家族などが「おすすめ」に表示された、ということで一部のユーザーの間で騒ぎになっていたのだが、この件をTwitter側に問い合わせた人がその顛末ややり取りしたメールを公開している。

これによると、Twitter側ではしばしば一時的に一部の人でアルゴリズムのテストや調整をしているそうで、グローバルIP情報がおすすめユーザに使用されたことは否定していない。ただ、 「おすすめユーザーのロケーションシグナルは、ある人には強すぎたり、匿名ユーザーの持ち主を発覚させる可能性があると学んだ。このテストは終了している。」ということで、今後採用される可能性は低そうである。

        twitter の ID により一連のコメントを集約して発言者を絞り込める形と「匿名で言いたいことが言える」というのは、やや整合性が取れないように見えますけど。
        実名か匿名かに関わらず ID を利用した結果、ID と紐づけられたパーソナリティーが形成されてしまう事となります。
        こうなると、「匿名によるコメント」と「ID 保有者によるコメント」という色合いの異なる部分が混じってきます。単純な例としては 47 氏とか、/.jp における ID とかが似た感じでしょうか。

        この結果、たとえ ID 自体は匿名で取得していて言いたいことを言っていても、Anonymous Coward や名無しさんとは異なる ID 保有者の発言となり、これはあなたが言っている「匿名で言いたいことを言える」とは違うものにならないでしょうか。

        twitter の場合だと ID を秘匿してコメントすることができれば「匿名で言いたいことを言える」は成立すると思いますが、現状 twitter はそのようなシステムではありません。/.jp や 2ch で AC や名無しさんによるコメントができないような作りだった、というのと大差ないと思います。

        そして、元コメントの「特定されて困るようなことを全世界に発信する神経が私には理解出来ません。」というのは、匿名性を持たせたいと利用者が思っている ID に対し、形成されているパーソナリティーの匿名性を ID 保有者自身で破壊する行為に他ならない訳ですから、サブジェクト通り「使い方がおかしい」という事にしかなりえないのではないでしょうか。

      AはおすすめされたユーザーBをフォローする。
      ↓
      BはAが身近な人であることを知らないままツイートし続ける。
      ↓
      Bの個々のツイートは個人を特定できるようなものではないが

      ↓
      AはBにカマをかけB=B'を確信する。
      ↓
      Aは何も知らないBのツイートを眺めてニヤニヤする。

      おすすめユーザーに出なくても、こういうことは起こりえますが
      おすすめユーザーに出ることで起きる確率が高くなることが考えられますね。
