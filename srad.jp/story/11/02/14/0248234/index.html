Windows 7/Server 2008 R2のSP1、2月17日より順次公開 41

ストーリー by hylom 2011年02月14日 12時01分
存在を忘れていた 部門より

Aluminum-Carbide 曰く、

Windows Server 2008 R2 SP1日本語版およびWindows 7 SP1 日本語版が2月17日より提供されるとのこと（日本マイクロソフトのニュースリリース）。

Windows Server 2008 R2 SP1では、仮想マシンからのグラフィックス画面転送機能「RemoteFX」や、メモリ利用の効率化により仮想マシンの集約率を高める「Dynamic Memory」などの新機能を実装しているとのこと。

ゲームはやらないがGPUをもてあましている人にはRemoteFXに使わせてみるのも案外面白そうかな？

  • いやいや、それほとんど意味ないでしょう (スコア:1)

    by Stealth (5277) on 2011年02月14日 14時44分 (#1902634)

    ゲームはやらないがGPUをもてあましている人にはRemoteFXに使わせてみるのも案外面白そうかな？

    DirectX の命令などをリモートで描画して結果をローカルに転送してきていたものを、リモートから API を飛ばしてきてローカルで描画するというもので、リモートホスト上で実行する動画、楽曲、ゲームなどをローカル GPU を利用して描画するための物です。
    ですから、今までローカルで GPU を持て余すような使い方をしているのに RemoteFX を使ったら GPU をバリバリ使います、なんていうのは色々考えにくいです。

    現状ではユーザーが Hyper-V などを利用してユーザーのデスクトップ環境を集約しているサーバーにリモートデスクトップで接続した際、自動的に該当仮想環境を起動して接続させる、といったシステムが存在しており (Windows Server 標準)、こうした形態で利用する場合に動画などを再生した場合のパフォーマンスを増大させることが主眼でしょう。
    Dynamic Memory も Hyper-V 仮想環境における割り当てメモリ量の可変化ですし、SP1 の基本的な強化点はこの辺りに集中しているだけではないでしょうか。
