Windows 8 では異なる 2 種類の UI が選択可能に 102

ストーリー by reo 2011年09月05日 12時30分
それならば居直れ 部門より

ある Anonymous Coward 曰く、

Windows 8 では新 UI「Metro」が導入されるが、それだけでなく従来のデスクトップも今まで通りに選択できるとのこと (CNET Japan の記事より) 。

しかし、Microsoft は UI について「妥協しない」と述べているが (Building Windows 8 の記事)、これは妥協そのものではないか、という指摘もある (TechCrunch の記事) 。

「Metro」は最新の Windows Phone などでも採用されている UI。タッチ操作には向いているかもしれないが、マウス操作で使いやすいか、業務での使用に適しているかは分からない。とりあえず、ユーザーを混乱させることがないように願いたい。

                > どうしてもクラシックメニューでないと困る

                選択可能な項目が一度に全部画面全体に表示されるので、どこに何があるかを覚えやすいのです。
                よく使う項目は、画面のどのあたりとか、上から何番目とか、そういう感覚で記憶してるので、
                Ctrl+Esc、P、↓、↓、↓、→、Enter、なんて具合に、脳が反射的に操作してる感じ。
                カーソルキーだけでほとんどのことができるので、直感的に操作できるのです。
                慣れの問題といわれれば、否定はしませんが。

                Vistaの標準スタートメニューは、Ctrl+Esc、↑、→で、“すべてのプログラム”に移れるので、
                その点は許しますが、なんといっても表示領域が狭いです。
                画面全体に全項目がパッと出てくれた方がいいです。
                それから、コントロールパネルは、Ctrl+Esc、Tab、↓、↓、↓とか押さなくちゃいけない。
                できたら、Alt+Cなどのアクセラレータが設定してあるとよかったのですけど。

                Vistaはともかく7のコントロールパネルはひどいです。表示モードの「詳細」が廃止されてしまいました。
                「アイコン」モードと「一覧」モードだけになりました。詳細モードの縦一列に並んでいるのが、
                探しやすくて好きだったのですが、残念です。
                これは、外部ファイル管理ソフト（こないだ出た秀丸ファイラーとか）に頼らなくてはなりません。（というか、それで解決できますが。）

                最近のUIは、総じて、マウス操作のことしか考えられていないような印象を受けます。キーボードが軽視されていると感じます。

                あと、リボンUIについてですが、先日のトピックでも書きましたが、リボン自体には文句はない（興味がないのであってもなくても構わない）のですが、
                メニューバー軽視には非常に憤りを感じます。

                # 将来は、マウス操作が軽視されて、タッチパネル依存のUIが増えてくるのでしょうかね？
