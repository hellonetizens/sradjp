Firefox 3.5リリース 148

ストーリー by soara 2009年07月01日 0時40分
Shiretoko Launch 部門より

あるAnonymous Coward 曰く、

ついにFirefox 3.5がリリースされました。目玉は以下の3つになると思います。

  • TraceMonkey JavaScriptエンジンによるパフォーマンスの向上
  • フリーなビデオコーデックOgg Theoraと音声コーデックOgg Vorbisのサポート
  • ユーザーの履歴をブラウザに残さないプライベートブラウジング機能の搭載

個人的にはOgg Theora/VorbisがWikipediaだけでなく、Youtubeやニコニコ動画でサポートされるかが気になります。

            • Firefox 更新時の拡張機能の互換性 (Re:Sage-Too が未対応なので) (スコア:4, 参考になる)

              by fcp (32783) on 2009年07月02日 7時25分 (#1597829) ホームページ 日記

              と言うか、そもそもの問題として。
              何故Firefoxの拡張処理系自体が互換性を維持しようとしない（できない）のでしょうか？

              どうして Firefox のバージョンが上がるたびにアドオンの作者が何かしないといけないのか、という疑問は僕もずっと持っていました。アドオンの互換性検査を省略して本来対応していないバージョンの Firefox で動かすという「ハック」があって、それで良いならもともと互換性検査なんてもの不要のはずですし。

              僕はアドオンの中でも拡張機能のことしかわからないので、以下は拡張機能についての話です。自分で拡張機能を書いて [mozilla.org]みてわかったのは、ある拡張機能が、 Firefox のバージョンを上げたり、他の拡張機能と同時に使ったりしても動くというのは、ほとんど偶然だということです。

              (ただし、他の拡張機能とうまく協調動作できるようにする工夫というのはちゃんとあって、すべてが偶然だというわけではありません。 Piro さんの「他のアドオンと衝突しないように心がけたいし、心がけて欲しい [sakura.ne.jp]」を参照。僕はこれを読んで、 Piro さんのソフトウェア設計のセンスに脱帽しました。)

              拡張機能とは、 Firefox に対するパッチみたいなものです。もともと Firefox は (というか Mozilla アプリケーションという枠組みを使って書かれたアプリケーションはすべて)、 C++ や JavaScript で書かれた「コンポーネント」を XUL、 CSS、 JavaScript 等の言語で書かれた「クローム」でつないで動いています。拡張機能は、クロームの部分をちょっと書き換えることで動きます (複雑な拡張機能だと、独自にコンポーネントを追加する場合もあります)。

              なので、 Firefox 本体のバージョンを上げたり他の拡張機能をインストールしたりすることで、パッチを当てる対象となるコードが変われば、パッチがうまく当たらなくなったり、パッチは当たっても意図したとおりに動作しなくなったりします。

              もちろん、 diff コマンドで作り patch コマンドで当てるパッチとは違って、拡張機能をサポートすることを目的の一つとして用意されている「XUL オーバーレイ」等の仕組みを使うので、 diff と patch よりはずっと安定して動作するのですが、それも程度問題です。

              僕は最初、 Firefox 3.5 Beta 4 用に拡張機能を書いたのですが、同じ拡張機能が Firefox 3.0.10 でも動くというのは自分で Firefox の側のコードを見て実際に動作を確認するまで信じられませんでした。たまたまパッチ対象のコードが 3.0.10 と 3.5 Beta 4 の間で変わっていなかったから動いたわけですが。

              拡張機能というものが動く仕組みを一度見れば、拡張機能を対応バージョン以外の Firefox で動かす「ハック」なんて、恐ろしくて使えません。

              将来は、 Firefox のバージョンを上げるたびにアドオンの互換性を気にしなくて済むような仕組みができると良いですね。

              (とは書いてみたものの、結局僕は自分の拡張機能について、対応する Firefox のバージョンを 3.5 までではなく 3.5.* まで (つまり 3.5.1 等にも対応している) と書いてしまっています。最初はよくわからずに指定していたのですが、今では「Firefox 3.0.10 から 3.5 まで変わらなかった部分なのだから、きっと 3.5.x で変わる可能性は低いだろう」と思ってそのままにしています。)
