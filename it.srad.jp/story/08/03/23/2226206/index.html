pinbou 曰く、

      ご心配なく。「Firefox 3 Memory Usage [pavlov.net]」という、記事が出たのは3月11日です。

      Parmenter氏はFirefox 3のメモリ管理の手法として、まず第一にメモリの断片化の解消を挙げている。氏は昨年11月にメモリの断片化に関するエントリ「Memory fragmentation [pavlov.net]」も投稿しているので、こちらとあわせて読むと参考になるだろう。

      そのほか、各種キャッシュのチューニングや、画像データの取り扱いなど、要点ごとに詳細な解説が掲載されている。また、Firefox 3 Beta 4とFirefox 2.0.0.12、Internet Explorer 7のメモリ使用を比較するベンチマークテストも行っている。

      ZDNet builder [zdnet.com] (03/13)

      とあるように、記事の主旨は、他ブラウザとの性能比較ではなく、Firefox 3でのメモリ使用量改善の取り組みの解説です。builderでは触れられていませんが、XPCOM cycle collector（循環参照に重点を置いたガベージコレクタ）の改善も奏功しているようですね。

      これがしばらくしてars technica [arstechnica.com] (03/17) 、Slashdot本家 [slashdot.org] (03/18) で取り上げられ（Safari 3.1のリリース直前）、またしばらくしてSlashdot Japan [srad.jp] (03/23) で今あなたに読まれていると。経路によっては話の伝わりが遅いというだけですよ。
