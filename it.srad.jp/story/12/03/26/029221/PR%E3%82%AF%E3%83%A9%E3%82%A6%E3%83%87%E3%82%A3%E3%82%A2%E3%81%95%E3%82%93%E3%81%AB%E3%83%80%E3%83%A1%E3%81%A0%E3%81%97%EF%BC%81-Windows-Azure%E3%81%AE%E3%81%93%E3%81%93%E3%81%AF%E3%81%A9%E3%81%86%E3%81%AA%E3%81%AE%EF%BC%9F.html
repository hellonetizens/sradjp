 PR:クラウディアさんにダメだし！　Windows Azureのここはどうなの？ 84

ストーリー by nabeshin 2012年03月26日 11時50分
帰ってきたクラウディアさん 部門より

Windows Azureが2010年2月に正式サービスを開始してから2年が経った。mixi Xmasやスクウェア・エニックスなどがWindows Azureを採用するなど、徐々に利用実績は増えてきているものの、最近ではさまざまなクラウドサービスが登場してきており、「クラウドといえばWindows Azure」という状況にはまだほど遠い。

では、なぜWindows Azureは受け入れられにくいのだろうか。その理由としては、やはりWindows Azureに対する不満があるからではないだろうか。そこで、Windows Azureをよく知るクラウディア・窓辺さんに、Windows Azureに対する不満をぶつけてみた。

クラウディアさんは好きだけど、Windowsは嫌いなんだよね。2000の頃にひどい目に遭ったし。

ソ、そんな昔のハナシを持ち出されても……。Azureで使っている最新の2008R2やIIS7.5は安定性も性能もだいぶ改善されているワ。

それに、いまどきクラウドのOSなんて気にしなくても、ファイルをFTPで更新できるようにする「Windows Azure Web Hosting Suite」とか、Ruby on Railsアプリをデプロイする「NougakuDoCompanion」といった便利ツールを使えばWindowsを気にすることなくPHPやRubyでの開発に専念できるのヨ。

最近ちょっとnode.jsにハマってる私のイチオシはブラウザだけで使えるIDEのCloud9ネ。あと、管理コンソールはSilverlightを使っているのでMacからでもアクセスできるわヨ。

Visual Studioがないと使えなさそうだし、Visual Studio高すぎ

デキる開発者なら当然MSDNサブスクリプションに加入しているわよネ？　えっ、入ってないの？　それは残念だけど、Windows 7にVisual Studioが入ってるPCがあればあとはWindows Azure Tools for Microsoft Visual StudioのインストールだけでOKヨ？　たったの約40MB！

え、Visual Studioなんてナイ？　それならMicrosoft Web Platform Installer 3.0で開発環境一式を簡単にインストールできるワ。ちょっとダウンロードとインストールに時間はかかるケドそれくらいは我慢シテネ。Visual Studio無償版でもいいけどできれば買ってもらってネ。

iPhoneやAndroidのアプリ向けには使えないよね？

大きな誤解ヨ。すでに多くのiOS/AndroidアプリがAzureのサービスを利用しているワ。ブラウザアプリならjQuery Mobileなどでフツウに作れるし、Objective-CやAndroid上のJavaアプリからストレージや認証、プッシュ通知サービスなどを便利に使えるToolkitもマイクロソフトが提供しているのヨ。

ネットの噂ではiPhone/iPadでみんな使ってるあのサービスもAzureを使っているトカ。ナイショだけどネ。

利用できるアプリやミドルウェアが少ない！

そんなことはないワ！　Windows AzureはWindows Serverをベースにしているから、Windows Serverで動くものならだいたい動く（ハズ）ヨ。WordPressなんかは楽勝ネ。

日本の動画サイトでよく使われてるFlash Media Serverのような製品もフツウに動くし、ほぼ同じ環境だから手元のWindows Serverで動作確認できればサポートの問題もほぼ回避できるワ。まだβだけどHadoopもJavaやJavaScriptで使えるし、簡単に集計レポートがグラフ化できるSQL Azure Reportingと組み合わせるとオモシロイ分析ができるかもネ。

ストレージやデータベースの仕様がワケ分からない！

そうね、確かにいろんなサービスがあるからちょっと複雑カモ。よく使うのはAmazonのS3に相当するBLOBかしら。仮想マシンからはWindows Azure Driveを使えば、Windows Azure上で普通のファイルシステムのように扱える永続的なストレージを利用できるワ。

あと、SQL AzureはMicrosoft SQL Serverが使えるならほとんど同じ感覚で使えるワヨ。ZendやCake、SymfonyなどのPHPフレームワークからもSQL ServerドライバでアクセスOK。MySQLも構築できるけどCPUとメモリの多い仮想マシンを使うより、容量課金で自動管理のSQL Azure使った方が金額的にもおトクかしら。

デプロイが遅い！　サーバーの起動も遅い！

確かにデプロイに十数分かかることはあるけど、自動管理や冗長性を確保するために必要な処理を全部裏でやってくれているんだからガマンしてネ。複数のインスタンスをつくると電源やスイッチも含めてSPOFが存在しないように配置してくれるのヨ。アプリを更新するときもインプレイスアップグレードという仕組みでサービスを止めることなく少しずつ自動的に適用してくれるし。

開発時など頻繁に更新しないといけないときは、静的コンテンツにしてBLOBに置くか、差分デプロイができるWeb配置ツールを使えば1分くらいで完了するので、これを使うのをオススメするワ。

ハーフのクラウディアさんはいいかもしれないけど日本語の情報が少ないのは苦痛。どこ見りゃいいのさ？

アナタ、私の活躍を見てナイの？　とりあえずここは必見ネ。ソーシャルアプリならここにまとまっているワ。ほかにもMSDNのWindows Azure 開発者向け技術情報に日本語ドキュメントがあって、オンラインのフォーラムもあるワヨ。実はStack Overflow（海外の技術者向け掲示板）でWindows AzureのQuestionは意外にタクサンあるので、これも参考になるワ。

日本だとユーザー会の活動も活発で、twitterでハッシュタグ #jazug をつけてつぶやくとAzure使いのみんなが助けてくれるカモ。女子部のみんなもがんばってるワ。

さすがクラウディアさん、Windows Azureに関する現状についてもしっかり分析しているようだ。このほかにも/.J読者の方でWindows Azureに対する不満点をお持ちの方がいたら、コメントでぜひ表明してほしい。まだ触ったことのなければ無料評価版などで突っ込みを。クラウディアさんが回答してくれるかも？
