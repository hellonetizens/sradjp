Yahoo! ジオシティーズ、HDD の故障による障害発生中。データ消失の可能性も 27

ストーリー by reo 2009年07月16日 11時30分
クレーム率を見て今後のサービス継続の可否を決定するとか 部門より

ある Anonymous Coward 曰く、

Yahoo! ジオシティーズからのお知らせによると、Yahoo! JAPAN の Web サイト無料ホスティングサービス「Yahoo! ジオシティーズ」で 7 月 13 日に障害が発生し、一部のページが表示されない不具合が起こっているとのこと。

不具合の原因は HDD の故障とのことで、ホスティングされている Web サイトのう約 95 % は現在は問題なく Web ページの閲覧・更新が可能であるが、一部のユーザーについては現在もサービスが利用できず、データが消失した可能性もあるようだ。

また、この障害の影響で Web サイトの新規開設や有料プランへの変更と行ったサービスが現在利用できなくなっているとのこと。

    • Re:インターフェイスの故障かなぁ・・ (スコア:2, 参考になる)

      by SteppingWind (2654) on 2009年07月16日 14時29分 (#1605975)

      インターフェイス障害は確かに滅多には起きないですけど, 起き得る可能性を無視できるほどは信頼性が高くなく, かつ一旦起こったときの障害規模が大きいですからね.

      これが汎用機や高価な大規模ストレージ装置なら, ハードウェアレベルの多重化や事前の障害検知が充実しているから構成/運用的には楽なんですけどね. IO性能がそれほどクリティカルではなく, CPUパワーに余裕があるなら, インターフェイスレベルで多重化したソフトウェアRAID1で構成するのも手ですね. PCサーバから中規模程度のUNIXサーバなら比較的廉価で実現できる手だと思います.

      個人的にはインターフェイス自体が壊れた経験はありませんが(仕事では1,2回ほど), SCSIに接続していた機器がいかれて, それに引きずられて芋づる式に複数のディスクでIOエラーが発生なんてことはありました.

        1)その枝にぶら下がってるディスクがまとめておかしくなるのがインターフェース故障。
        2)その箱にぶら下がってるディスクがまとめておかしくなるのがシェルフ故障。
        3)単発でぽろりぽろりディスクがおかしくなるのがディスク故障。
